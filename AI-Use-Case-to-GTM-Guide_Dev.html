BA Genie Write-Up Sample

This sample shows the expected level of technical documentation for a picked-up use case before lab-owner sign-off.

Solution Summary

BA Genie is an AI-powered requirements accelerator that converts business inputs such as meeting recordings, transcripts, Word documents, emails, PDFs, and supporting files into structured delivery artifacts including epics, features, user stories, and test cases. The solution keeps a human reviewer in the loop before any artifact is finalized or pushed into a project management tool such as Jira.

1. Models Used Within the Flows

  • Video-to-text / speech-to-text model: used to transcribe uploaded meeting recordings or audio files into editable transcript text.
  • Primary large language model: used to generate epics, features, stories, and test cases from approved transcript and document inputs. Sample implementation: Anthropic Claude 3 Sonnet via AWS Bedrock.
  • Secondary / cost-optimized model option: used as an approved fallback where cost sensitivity matters more than output richness. Sample implementation: Mistral-7B via AWS Bedrock.

The flow is: upload source material, transcribe if needed, review transcript, generate requirement artifacts, run orchestrator consistency checks, complete human review, then publish approved work items into Jira.

2. Frameworks Used to Implement Agents

  • Application runtime: Node.js
  • Agent orchestration framework: LangChain
  • Deployment packaging: Docker container image
  • PM tool integration: Jira API

This aligns to BA Genie technical material that shows a sample Node.js / LangChain architecture, containerized deployment, and integration with Jira or Azure DevOps.

3. Human or Panel-of-Agents Decision Points

Human decision points

  1. Decide whether the uploaded content is sufficient to proceed.
  2. Review and approve the transcript created from audio or video.
  3. Select the artifact type to generate: epic, feature, user story, or test case.
  4. Review and refine the generated output.
  5. Approve whether the artifact is ready to be published into Jira.

Agent / orchestrator decision points

  1. Route input through transcription, direct parsing, or both.
  2. Extract relevant content and remove irrelevant meeting noise.
  3. Identify candidate epics, features, stories, or test cases.
  4. Enrich outputs with business value, scope, dependencies, and regulatory context.
  5. Run orchestrator-led consistency and quality checks before output is accepted.

4. How Key Workflows Are Tested

  • Validate supported inputs: audio, video, Word, PDF, email, and spreadsheet files.
  • Verify transcript generation, human edits, and transcript approval flow.
  • Test generation of new epics, features, user stories, test cases, and updates to existing requirements.
  • Validate Jira publication, field mapping, and issue linking.
  • Benchmark output quality for completeness, structure consistency, hallucination risk, duplication risk, and regulatory relevance.
  • Confirm no output is published without human approval.
  • Run integration, performance, security, and observability checks before release.

5. How the Solution Will Be Deployed

  • Deployment type: containerized solution accelerator
  • Sample cloud target: AWS
  • Container runtime: ECS
  • Storage: S3 for source content and generated artifacts
  • Model access: AWS Bedrock
  • Monitoring: CloudWatch or client-approved monitoring stack

Capco provides the deployable BA Genie container image. The client environment hosts the containers, configures access to Bedrock, storage, and Jira, and then promotes the solution from pilot to broader usage once testing and governance checks are complete.
